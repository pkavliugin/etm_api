# Excel Upload Service

Минимальный FastAPI backend для загрузки и обработки Excel файлов.

## Требования

- Python 3.9+
- FastAPI
- pandas
- openpyxl
- requests

## Установка

```bash
# Создать виртуальное окружение (рекомендуется)
python -m venv venv
source venv/bin/activate  # на Windows: venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt
```

## Конфигурация ETM API

Перед запуском приложения необходимо настроить учетные данные ETM API в [main.py](main.py):

```python
ETM_LOGIN = "your_login"           # Замените на ваш логин
ETM_PASSWORD = "your_password"     # Замените на ваш пароль
ETM_API_BASE_URL = "https://ipro.etm.ru/api"  # Production
# ETM_API_BASE_URL = "https://itest2.etm.ru/api"  # Test (раскомментируйте для тестирования)
```

**Авторизация:**
- Метод: `POST /user/login`
- Session ключ действует **8 часов**
- После истечения времени ключ будет обновлен автоматически

**Ограничения по частоте запросов ETM:**
- Авторизация: 1 запрос в 2 минуты
- Характеристики товара: 1 запрос в секунду
- Цены: 1 запрос в секунду
- Остатки: 1 запрос в секунду

## ETM API - Справочная информация

### API документация

- **Production API:** https://ipro.etm.ru/api/v1
- **Test API:** https://itest2.etm.ru/api/v1

### Основные методы:

1. **Авторизация** `POST /user/login`
   - Параметры: log (логин), pwd (пароль)
   - Возвращает: session ключ (действует 8 часов)

2. **Характеристики товара** `GET /v2/goods/{id}`
   - Получает полную информацию о товаре
   - Параметры: id (код), type (etm/cli/mnf), session-id

3. **Цены** `GET /v1/goods/{id}/price`
   - Получает текущие цены товара
   - Параметры: id, type, session-id

4. **Остатки** `GET /v1/goods/{id}/remains`
   - Получает остатки на складах
   - Параметры: id, type, session-id

## Функции обработки данных

### normalize_name(name: str) → str

Нормализует название товара перед поиском в ETM:
- Приводит к нижнему регистру
- Заменяет кириллицу 'х' на 'x'
- Удаляет лишние символы (скобки, запятые, кавычки и т.д.)
- Убирает дополнительные пробелы

**Пример:**
```python
normalize_name("Товар (х)") → "товар x"
normalize_name("Ноутбук, сер.") → "ноутбук сер"
```

Функция используется перед отправкой названия в `search_etm`.

### search_etm(name: str) → Dict

Поиск товара в ETM API с **встроенным кэшем**:
- Автоматически нормализует названия перед поиском
- **Проверяет кэш** - если запрос уже вызывался, вернет сохраненный результат
- Если результата нет в кэше, делает HTTP запрос
- Сохраняет результат в кэш для будущих запросов

Кэш сохраняется в памяти приложения (очищается при перезагрузке сервера)

## API Endpoints

### GET /
Главная страница с веб-интерфейсом для загрузки Excel файлов.

**Доступ:** `http://localhost:8000`

**Интерфейс:**
- Форма загрузки файла (Drag & Drop)
- Кнопка "Обработать" 
- Автоматическое скачивание результата
- Визуальное отображение статуса обработки

### GET /health
Endpoint для проверки здоровья сервиса.

### GET /search/{product_name}
Поиск товара в ETM по названию.

**Параметры:**
- `product_name` (path): Название товара для поиска

**Пример запроса:**
```bash
curl http://localhost:8000/search/Ноутбук
```

**Пример ответа (успех):**
```json
{
  "query": "Ноутбук",
  "results": {
    "id": "123",
    "name": "Ноутбук ASUS",
    "price": 45000
  }
}
```

**Возможные ошибки:**
- **400** - пустое название товара
- **503** - недоступен ETM API

### POST /upload
Загрузка и обработка Excel файла.

**Параметры:**
- `file`: Excel файл в формате .xlsx

**Требуемые колонки в файле:**
- "Наименование" - название товара/услуги
- "Количество" - количество (целое число)

**Поведение:**
1. Читает Excel файл
2. Для каждой строки выполняет поиск товара в ETM API (функция `search_etm`)
3. Возвращает Excel файл с колонками:
   - `name` - название товара
   - `quantity` - количество
   - `etm_result` - результат поиска в ETM API (JSON в ячейке)

**Пример ответа:**
Возвращается Excel файл (application/vnd.openxmlformats-officedocument.spreadsheetml.sheet) с данными:

| name      | quantity | etm_result                      |
|-----------|----------|--------------------------------|
| Товар 1   | 10       | {"id": "123", "price": 5000}   |
| Товар 2   | 5        | {"id": "124", "price": 3000}   |

Файл автоматически скачивается с именем `result.xlsx`

**Возможные ошибки:**
- **400** - некорректный формат файла, отсутствуют необходимые колонки
- **500** - внутренняя ошибка сервера

## Обработка ошибок

- **400** - некорректный формат файла, отсутствуют необходимые колонки, ошибка при обработке данных
- **500** - внутренняя ошибка сервера

## Тестирование с curl

```bash
# Загрузить файл и сохранить результат
curl -X POST -F "file=@example.xlsx" http://localhost:8000/upload -o result.xlsx
```

## Тестирование с Python

```python
import requests

files = {'file': open('example.xlsx', 'rb')}
response = requests.post('http://localhost:8000/upload', files=files)

# Сохранить полученный Excel файл
with open('result.xlsx', 'wb') as f:
    f.write(response.content)
    
print("Файл успешно сохранен как result.xlsx")
```
