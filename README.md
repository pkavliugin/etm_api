# ETM Excel Processor - Профессиональная обработка спецификаций

Полнофункциональное FastAPI приложение для обработки Excel спецификаций с интеграцией API ЭТМ. Автоматически обогащает данные товарами, ценами, артикулами и информацией о наличии.

## 🎯 Цель приложения

- **Обработка Excel спецификаций товаров**
- **Получение актуальных данных из API ЭТМ:**
  - Цен на товары
  - Информации о наличии и сроках поставки
  - Артикулов товаров
- **Формирование структурированного Excel файла**

**Результат используется для:**
- Расчета коммерческих предложений
- Планирования закупок материалов
- Анализа ценовой политики поставщиков

## ✨ Ключевые возможности

- 🚀 **Быстрый запуск**: Один клик в VS Code
- 🎨 **Современный веб-интерфейс**: Drag & drop загрузка файлов
- ⚡ **TTL кэширование**: 1 час кэша для оптимизации запросов
- 🎯 **Умный поиск**: Fuzzy matching для точного подбора товаров
- 📊 **Множественный выбор**: Автоматический выбор лучшего товара из результатов
- 🔄 **Обработка ошибок**: Graceful handling всех типов ошибок
- 📈 **Расширенный вывод**: Артикулы, единицы измерения, цены, наличие

## 📊 Формат входных данных

Excel файл должен содержать колонки:
- **Наименование** (обязательно): Название товара
- **Количество** (обязательно): Количество единиц

### Пример входного файла:

| Наименование      | Количество |
|-------------------|------------|
| Транзистор 2N2222 | 10         |
| Резистор 10кОм    | 5          |
| Конденсатор 10мкФ | 3          |

## 📤 Формат выходных данных

Результат - Excel файл со следующими колонками:

| Исходное название | Найденное название | Артикул | Единица измерения | Цена за единицу | Наличие / срок | Статус |
|-------------------|-------------------|---------|-------------------|-----------------|----------------|--------|
| Транзистор 2N2222 | Транзистор 2N2222A | 2N2222 | шт | 5.50 | 10 | ok |
| Резистор 10кОм    | Резистор 10кОм 0.25Вт | R10K | шт | 0.15 | 5 | ok |
| Конденсатор 10мкФ | Конденсатор 10мкФ 16В | C10uF | шт | 2.30 | 0 | ok |

### Логика выбора товара

1. **Поиск**: Fuzzy matching по названию (порог 70%)
2. **Приоритет**: Сначала товары с наличием
3. **Цена**: Среди доступных - минимальная цена
4. **Альтернативы**: Топ-3 варианта сохраняются в отдельном поле

Функция используется перед отправкой названия в `search_etm`.

### search_etm(name: str) → Dict

Поиск товара в ETM API с **встроенным кэшем**:
- Автоматически нормализует названия перед поиском
- **Проверяет кэш** - если запрос уже вызывался, вернет сохраненный результат
- Если результата нет в кэше, делает HTTP запрос
- Сохраняет результат в кэш для будущих запросов

Кэш сохраняется в памяти приложения (очищается при перезагрузке сервера)

## API Endpoints

### GET /
Главная страница с веб-интерфейсом для загрузки Excel файлов.

**Доступ:** `http://localhost:8000`

**Интерфейс:**
- Форма загрузки файла (Drag & Drop)
- Кнопка "Обработать" 
- Автоматическое скачивание результата
- Визуальное отображение статуса обработки

### GET /health
Endpoint для проверки здоровья сервиса.

### GET /search/{product_name}
Поиск товара в ETM по названию.

**Параметры:**
- `product_name` (path): Название товара для поиска

**Пример запроса:**
```bash
curl http://localhost:8000/search/Ноутбук
```

**Пример ответа (успех):**
```json
{
  "query": "Ноутбук",
  "results": {
    "id": "123",
    "name": "Ноутбук ASUS",
    "price": 45000
  }
}
```

**Возможные ошибки:**
- **400** - пустое название товара
- **503** - недоступен ETM API

### POST /upload
Загрузка и обработка Excel файла.

**Параметры:**
- `file`: Excel файл в формате .xlsx

**Требуемые колонки в файле:**
- "Наименование" - название товара/услуги
- "Количество" - количество (целое число)

**Поведение:**
1. Читает Excel файл
2. Для каждой строки выполняет поиск товара в ETM API (функция `search_etm`)
3. Возвращает Excel файл с колонками:
   - `name` - название товара
   - `quantity` - количество
   - `etm_result` - результат поиска в ETM API (JSON в ячейке)

**Пример ответа:**
Возвращается Excel файл (application/vnd.openxmlformats-officedocument.spreadsheetml.sheet) с данными:

| name      | quantity | etm_result                      |
|-----------|----------|--------------------------------|
| Товар 1   | 10       | {"id": "123", "price": 5000}   |
| Товар 2   | 5        | {"id": "124", "price": 3000}   |

Файл автоматически скачивается с именем `result.xlsx`

**Возможные ошибки:**
- **400** - некорректный формат файла, отсутствуют необходимые колонки
- **500** - внутренняя ошибка сервера

## 🐛 Обработка ошибок

Приложение надежно обрабатывает все типы ошибок:

- **400 Bad Request**: Некорректный файл, отсутствующие колонки
- **401 Unauthorized**: Проблемы с авторизацией ETM API
- **503 Service Unavailable**: ETM API недоступен или превышены лимиты
- **500 Internal Server Error**: Внутренние ошибки приложения

Все ошибки логируются и возвращают понятные сообщения пользователю.

## 🧪 Тестирование

### Тестирование с curl

```bash
# Загрузить файл и сохранить результат
curl -X POST -F "file=@specifications.xlsx" \
     http://127.0.0.1:8000/upload \
     -o result.xlsx
```

### Тестирование с Python

```python
import requests

# Загрузка файла
files = {'file': open('specifications.xlsx', 'rb')}
response = requests.post('http://127.0.0.1:8000/upload', files=files)

# Сохранение результата
with open('result.xlsx', 'wb') as f:
    f.write(response.content)

print("✅ Результат сохранен как result.xlsx")
```

### Создание тестового файла

```python
import pandas as pd

# Создание тестовых данных

    'Количество': [10, 5, 3]
}

df = pd.DataFrame(data)
df.to_excel('test_specifications.xlsx', index=False)
print("✅ Тестовый файл создан")
```
